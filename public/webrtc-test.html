Stage 4 · WebRTC File Transfer

Peer-to-peer file transfer over DataChannel

This page supports string messaging and chunked file transfer on the same WebRTC DataChannel. Open it in two browser windows, connect the peers, choose a file, and send it. The receiver rebuilds the file locally and exposes a download link after reassembly completes.

Quick Steps

  1. Open this page in two browser windows.
  2. Use a different deviceId in each window.
  3. Click Connect Signaling on both sides.
  4. On side A, enter side B's deviceId.
  5. Click Start WebRTC on side A.
  6. Wait until the DataChannel status changes to open.
  7. Choose a file and click Send File.
  8. Wait until the receiver reaches 100% and use the download link.

Transfer Notes

  1. Files are split into fixed-size chunks before sending.
  2. Each binary chunk carries a small header with the chunk index.
  3. The receiver caches chunks in memory and rebuilds a Blob when complete.
  4. The current demo handles one active incoming file transfer at a time.
